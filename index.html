01 — About

私たちは、 小さく強い テクノロジーチームです。

東京を拠点に、Web 制作・AI 活用・DX 支援を組み合わせて、 課題そのものを再定義する仕事をしています。 「速く、美しく、本質的に」 が私たちの行動指針です。

03 — Culture

ここで働くことは、 毎日「学べる」 ということ。

年功序列ではなく実力主義。 若手にも責任ある仕事を任せる。 それが NOVA TECH のスタンダードです。

07 — Design Intent

このサイトは、 採用課題を解決する LP として設計しました。

「見栄えが良い採用サイト」 ではなく、 応募数・応募者の質・エントリー完了率 を改善するために何ができるかを起点に組んだ構造です。

INTENT / 01

ファーストビューで「迷わせない」

課題仮説 採用 LP の離脱の多くは ファーストビュー 3 秒以内 で発生。 「何の会社か」「自分に関係があるか」 が即座に伝わらないと読まれない。

打ち手 ヒーローを 「事業内容＋募集中の事実＋数値で語る指標」 の三層に圧縮。 視線が止まる前に判断材料を提示する設計に。

INTENT / 02

不安を順番に解消する縦動線

課題仮説 応募者が抱える不安は 「合うか／働けるか／応募して大丈夫か」 の3層。 順番を間違えると、 価値訴求の前にフォームを閉じられる。

打ち手 About → Service → Culture → Work Style → Interview → Recruit → FAQ → Contact の順で、 「会社理解 → 自分との接続 → 具体条件 → 行動」 を構造化。

INTENT / 03

CTA を 11 箇所、 階層を分けて配置

課題仮説 CTA が少ないと離脱、 多すぎると意思決定が鈍る。 また 「すべて同じボタン」 では押す動機にならない。

打ち手 プライマリ ／ セカンダリ ／ ステルスの 3 階層 に分け、 各セクション末尾を 「次に進む自然な場所」 として CTA を配置。

INTENT / 04

スマホ離脱を構造で止める

課題仮説 採用 LP のアクセス比率は モバイルが 6 ～ 7 割。 スマホでの読みづらさ・操作しづらさは、 そのままエントリー機会の損失に直結する。

打ち手 モバイルファーストで設計。 タッチターゲット 44px 以上、 ハンバーガー UI、 縦積みフォーム想定など、 「片手で完結できる」 ことを基準に組み立て。

KPI Targets

公開後 3 ヶ月で目指す数値

各指標は Google Analytics 4 ／ ヒートマップツール (Microsoft Clarity 等) ／ A/B テストで継続計測することを想定しています。

15%+ 応募ボタン CTR
ヒーロー直下のプライマリ CTA が押された比率。 ファーストビューの訴求力を測る最初の指標。
8%+ エントリーフォーム到達率
セッションのうち Contact セクション ／ エントリーリンクに到達した割合。 動線設計の有効性を測る。
35%+ FAQ 閲覧率
FAQ のアコーディオン展開率。 不安解消ニーズの強さと、 応募一歩前の検討の深さを示す。
50%- モバイル離脱率
モバイルセッションのうち、 ヒーローのみで離脱した比率。 50% 未満をベンチマークに設定。
2分+ 平均滞在時間
採用 LP として 「読まれている」 状態の目安。 競合比較なしで 2 分超を維持できれば想定通り。
90+ PageSpeed (Mobile)
表示速度はそのまま離脱率に直結する。 Lighthouse Mobile スコア 90 以上を継続維持する。

◇ 数値はサイト公開後の 仮説目標値 です。 計測結果に応じて、 ヒートマップ ／ A/B テストでコピー・配色・配置を継続改善することを前提に設計しています。

運用・改善のしやすさ

  • ◆

    WordPress 化を前提とした構造

    各セクションを独立した <section> として分離。 募集職種は details 要素で 1 件ずつ独立しており、 カスタム投稿タイプへの移行が容易です。

  • ◆

    CSS 変数で配色を一元管理

    ブランドカラーやタイポグラフィは :root に集約。 「青を少し明るく」 のような調整が 1 行 で完結します。

  • ◆

    FLOCSS／BEM 風の命名規則

    l- ／ c- ／ p- ／ u- のプレフィックスで役割を分離。 他のメンバーが触っても破綻しないコード設計にしています。

  • ◆

    テキストと構造を完全分離

    募集要項の追加 ／ 修正は HTML のテキストノードを差し替えるだけ。 デザインを崩さずに採用情報をアップデートできます。

08 — FAQ

よくあるご質問

応募前に多くの方が気になる質問をまとめました。 ここに無い質問は、お気軽にお問い合わせください。

  • Q. 未経験ですが応募できますか？ +

    はい、可能です。 Frontend Engineer と UI / UX Designer は未経験歓迎ポジションです。 大切なのは「現時点での技術力」ではなく、 「学び続けられること」と「何かしらの制作物があること」です。 簡単な HTML / CSS の制作物や、 個人で作った Figma のデザインで構いません。

  • Q. 選考フローを教えてください。 +

    ① カジュアル面談 (30 分) → ② 一次面接 (60 分) → ③ 課題提出 → ④ 最終面接 (60 分) → ⑤ 内定。 平均 2 〜 3 週間程度で進めています。 状況に応じて短縮も可能です。

  • Q. どんな開発環境を使っていますか？ +

    エディタは VS Code、 バージョン管理は GitHub、 コミュニケーションは Slack、 ドキュメントは Notion を中心に使っています。 機材は希望に応じて MacBook Pro または Windows ノートを支給します。

  • Q. フルリモートでも問題ないですか？ +

    はい、ほとんどのメンバーがフルリモートで働いています。 全国どこからでも応募可能です。 月 1 回任意参加の「チームデイ」のみオフィス開催ですが、 こちらもオンライン参加可能です。

  • Q. ポートフォリオは必須ですか？ +

    必須ではありませんが、 何かしらの制作物・成果物があると選考がスムーズです。 完成度より「何を考えて作ったか」を重視します。 GitHub のリポジトリ、 デザインカンプ、 個人ブログなど形式は問いません。
